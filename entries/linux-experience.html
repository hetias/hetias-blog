The Linux Experience

i've been using linux for almost 3 years now, i can't really believe it. Mostly because it never felt like that much of a big change and nowdays going back to windows might feel weird 'cause i would see all those problems that made me move to linux.

Now, to say that linux it's perfect or objectivly better than Windows or MacOs would be a plain up lie. It does has issues but i would argue these are more tolerable than whatever Windows and MacOs will do to you

Why to do it?

There's something that it's hard to deny about linux, and it's about being more technical than other majors OS's. Not to say that other OS's cannot be it, but it may look more developer friendly at first, especially knowing how hackable and customizble the OS can be. You really can change whatever you want on it.

Being a programmer that was my reason to move here as i thought it would be a more friendly experience in that sense. And i can't deny that at all, if you're a developer linux can be smoother sometimes. But this can make the transition harder for non technical people that may not care about the intricacies of the OS, every time i see a problem my mind goes "Oh! let's see how to fix this"-mentality as i like to learn how the Operative System works and does stuff. But that's just me, and surely after solving the problem once i may not be so happy if it happens again. For someone non technical this is a horrible experience and linux wouldn't fix itself.

Another reason was speed as i had a very slow laptop at this time and linux usually it's presented as more lightweight than Windows in general. I can't say it's faster but surely it's not slower than windows, but as time passes it's not rare to see benchmark between Windows and Linux where linux tends to be at pair or slightly better.

Not perfect at all

It's wonderful and at the same time atrocious how diverse the linux ecosystem is. Desktop enviroments, custom kernel versions, compositors, tiling windows managers and so much more that one gives for granted in Windows but here we can go full bananas with.The problem usually lies when you open apps that may not respect the same desktop enviroment design or may they not work because these expect you to use another linux distro and so much more. This diversity it's what makes linux special but at the same time hard to adopt, because besides the visual stuff if the actual important stuff doesn't work then we are in problems.

An OS it's just a bunch of software that let you use other software without worrying about managing hardware. Linux it's just a kernel which it's just the heart of the OS, but there so much more software that makes the whole OS to work. A distro it's a different way to manage that extra software, from custom kernels, alternative software options, and visual changes. To choose a distro it's to choose a class in an RPG, at first it's not that important but later on you may struggle.
All my homies hate Canonical, but Ubuntu works and besides their default Desktop enviroment it's more user friendly than others. I would rescommend Fedora tho (homies doubt about Red Hat too) as very stable and really the influence that Red Hat has over it it's not as bad as Ubuntu with Canonical.
Then we have to God distros, Arch and Debian, the neckbeards and the beardnecks. If you're new, just use Fedora for homework and Arch to break stuff (It's just one of those days when you lose a backup!).

If you want to get software on Windows you usually download an installer from the internet, run it and it's done (THIS IS UNSAFE AS FUCK EVEN THO WE HAVE IT NORMALIZED). On linux you usually use a package manager that stores software and manages dependencies related to these. I don't really know if you can break windows the same way you can brake linux with dependencies problem, i think not but besides, breaking your system it's a huge deal in linux even if it may be rare. Let's say you have package A (let's say a package it's just like a rar version of the binarie) and a package B, A depends on B and so if you want to install A, B has to be installed first so A can work properly. So, what if something goes wrong and B it's not correctly installed even if A got installed? Well, A it's not going to work. Right now i have 1190 packages in my system most of which may just be libraries that let other user program work, if i try to update but one really important library for reason doesn't update correctly we're fucked. This happen not so along with an update on Manjaro where a library (i think it was libxml?) failed on update so it just partially updated my system, which led firefox and emacs unusable. You can do a partial update if you know what you're doing, but most of the time it's better to do a full update at the right time and then check if everything it's still working, and if not timeshift can save lives. Always backup your system before updating!

Gaming it's another big factor on play here.
I've never been unable to play something in linux but that's because i pretty much play games through steam and never had to deal with too aggressive anti-cheats. Games like battlefield, fortnite, League of Legends and other multiplayer and competitive games use anticheats and specially at the kernel level. This makes it impossible to run through wine and as it will plantly not work. But if you don't play any of these games (as i don't) you're probably up to go. In any case this website collects info about games and their compatibility with steam proton, which is just valve's flawored wine, check the games you play the most on there and you will see if these are playable or not.
